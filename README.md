# Todo List Frontend — Vue

## Visão Geral

Este projeto consiste no desenvolvimento de uma aplicação frontend de lista de tarefas (Todo List) utilizando **Vue**.

A aplicação deve permitir que o usuário gerencie tarefas de forma simples e intuitiva, simulando um cenário real de aplicação web focada em organização pessoal e produtividade.

O foco do projeto está na construção de uma interface funcional, bem estruturada e de fácil manutenção, utilizando boas práticas do ecossistema Vue.

---

---

## Escopo do Projeto

### Incluído no escopo
- Exibição de uma lista de tarefas
- Criação de novas tarefas
- Edição de tarefas existentes
- Marcação de tarefas como concluídas
- Remoção de tarefas
- Persistência de estado (local ou remoto)

### Fora do escopo
- Sistema de autenticação de usuários
- Controle de permissões ou perfis
- Funcionalidades avançadas de colaboração
- Sincronização em tempo real entre múltiplos usuários
- Backend completo

---

## Requisitos Funcionais

- A aplicação deve permitir a criação de uma tarefa informando, no mínimo, um título.
- A aplicação deve exibir a lista de tarefas cadastradas.
- A aplicação deve permitir editar uma tarefa existente.
- A aplicação deve permitir marcar uma tarefa como concluída ou não concluída.
- A aplicação deve permitir remover uma tarefa da lista.
- A interface deve refletir imediatamente as alterações de estado.

---

## Requisitos Não Funcionais

- O código deve estar organizado de forma clara e legível.
- A aplicação deve utilizar uma estrutura de componentes bem definida.
- O projeto deve conter instruções básicas para execução local.
- A interface deve ser responsiva ou minimamente utilizável em diferentes tamanhos de tela.

---

## Tecnologias Sugeridas

**Obrigatórias**
- Vue — framework principal da aplicação
- HTML, CSS e JavaScript

**Opcionais**
- TypeScript
- Pinia ou outra solução de gerenciamento de estado
- Vue Router (se necessário)
- Bibliotecas de estilização (CSS Modules, Tailwind, etc.)
- Mock de API ou integração com backend simples

---

## Critérios de Conclusão

O projeto será considerado concluído quando:
- Todas as funcionalidades descritas nos requisitos funcionais estiverem implementadas.
- A aplicação puder ser executada localmente sem erros.
- O usuário conseguir gerenciar completamente suas tarefas pela interface.
- O repositório contiver um README com instruções claras de execução.
